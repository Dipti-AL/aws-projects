Lab 162: My Journey Building an RDS Server and Connecting via EC2I just finished a challenge lab where I had to set up an Amazon RDS database from scratch and get it talking to a Linux server. It was a great exercise in networking, security, and a bit of unexpected troubleshooting with Docker.Step 1: Provisioning the RDS InstanceFirst, I had to launch the database. I went with a MySQL engine. There were some specific lab restrictions I had to follow to keep things within the free tier/dev environment:Instance Class: I chose a burstable db.t3.micro.Storage: I set it to 100 GB using General Purpose SSD (gp2).Networking: I made sure it was launched inside the Lab VPC.Monitoring: I had to remember to disable Enhanced Monitoring in the additional configuration settings.![1.png] (Database creation success screen) I made sure to save my credentials (username: admin, password: lab-password) and the endpoint URL, since I'd need those to log in later.Step 2: Fixing the Security GroupThe database was up, but it was locked down. I had to go into the Security Group and add a new inbound rule.Type: MySQL/Aurora Protocol: TCP Port: 3306 Source: I set this to allow my LinuxServer to connect.![2.png] (Adding the 3306 rule to the Security Group) Step 3: Dealing with the "Authentication Plugin" ErrorThis is where it got tricky. I logged into my Linux server via Putty and installed the default MySQL client. However, when I tried to connect to the RDS endpoint, I got a nasty error: ERROR 2059 (HY000): Authentication plugin 'caching_sha2_password' cannot be loaded.Basically, the MySQL 8 database uses a new security plugin that my old Linux client didn't understand. Instead of messing with broken libraries on the server, I used a Docker container to run a modern MySQL 8 client.The commands I used:Bashsudo yum install -y docker
sudo service docker start
sudo usermod -a -G docker ec2-user
# After a quick relog...
sudo docker run -it --rm mysql:8 mysql -h [my-endpoint] -u admin -p
Step 4: Interacting with the DataOnce I was finally in, I switched to my database (USE mydb) and started building.Creating the TablesI created two tables: RESTART (for student info) and CLOUD_PRACTITIONER (for cert dates).![3.png] (Creating tables and inserting 10 rows of data) Running an Inner JoinTo wrap things up, I ran an INNER JOIN to see which students from the RESTART table had actually finished their Cloud Practitioner certification.![4.png] (The final join result showing Alice, Charlie, Eva, Grace, and Ian) What I Learned:RDS Setup: How to navigate the AWS console to spin up a managed DB.Security: The importance of matching port 3306 in security groups.Troubleshooting: Using Docker is a lifesaver when you run into "version hell" with OS packages.
