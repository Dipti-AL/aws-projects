**AWS SimuLearn Activities – Networking, Security and File Systems**

As part of the AWS re/Start program, I completed SimuLearn activities focused on networking, security, and storage. These labs helped me understand how things work in AWS by actually using the services instead of just reading about them.

**Networking Concepts**
In this section, I learned how to set up a Virtual Private Cloud (VPC) and organize resources inside it. I worked with subnets, route tables, and gateways to control how traffic moves.

I understood the difference between public and private subnets and how internet access is managed using Internet Gateways and NAT Gateways. I also practiced using Security Groups and Network ACLs to control traffic and improve security.

Overall, this helped me understand how AWS networking is structured and how to build a basic secure setup.

**Core Security Concepts**

In the security labs, I worked with IAM (Identity and Access Management). I created users, groups, and roles, and assigned permissions using policies.

I learned about the principle of least privilege, which means giving only the access that is needed. I also explored Multi-Factor Authentication (MFA) and how roles are used instead of sharing credentials.

Another important concept was the shared responsibility model, where AWS handles infrastructure security, and users are responsible for securing their own resources.

**File Systems in the Cloud**

In this part, I explored different storage options in AWS. I worked with object storage and basic file storage concepts.

I learned the differences between object storage, block storage, and file systems, and when to use each. I also understood how cloud storage is scalable and durable without needing manual setup.

**Overall Learning**

These SimuLearn activities helped me connect theory with practice. I was able to actually configure services and see how they work together.

Overall, this gave me a good foundation in networking, security, and storage in AWS, and helped me feel more confident working in a cloud environment.
